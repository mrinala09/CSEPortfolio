Problem

People struggle with cooking and are unable to follow recipes with long and complicated instructions, or are unable to decide on what to make with the items they have. This leads many people to order in rather than using a healthier alternative. With this app, we provide a perfect solution to this problem.

This problem doesn't just affect everyday adults, but students that go to school too. Often students need to make lunch for themselves in the morning to take to school. But they usually have trouble with this or are not sure how to cook. Without a proper and nutritious lunch, students are not able to perform properly in school and this causes a lot of problems for them making their day hard. Students most being adolescents, want something tasty, but it is best if it is also very nutritious.

Though some solutions already exist such as a manual physical cookbook, those are very inefficient and require effort and time to look through them and find your recipe. We want to make something that is easy for the user to use and requires minimal effort and provides instructions and fulfills cravings.

Solution Planning

Solution Planning

Brainstorming: Our General Idea was to make a program that was essentially a CookBook, except the user could define the ingredients they wanted to use.

API Databases/Pulling Data: Now we had an Idea on what to create, we first need a Database that contains all the recipes and ingredients, and a code that pulls recipes based on the user input.

UI with MVC and Flask: Our Final step was to create the UI for the user, since we are creating a web program, we need to use Flask and MVC to create a website and access servers.

Recipe App: Our final product is our recipe app that is a web-based program where a user can input ingredients they have and they will receive multiple recipes on what they can cook with it.

Solution Summary

Solution Summary

Sprint 1: Find an API Database and set it up, luckily a web API that contained a food and recipe database already existed, so we used that as our basis.

Sprint 2: Set up a function structure. We are going to make this program using Object Oriented Programming, and connect it all with the API database.

Sprint 3: Program the functions, Main: asks the user to list ingredients they want, fetch_recipe: fetches the recipes and the ingredients needed from the Edamam API, Display_recipe: Displays the dish, the index number, and the ingredients needed, and a recipe link.

Sprint 4: Create a User Interface using Flask and MVC so that this program can be run on a website.
